## The Core Insight

I built something I didn't understand, and it was correct.

This is not supposed to happen. We're taught that understanding precedes creation. You learn the theory, then implement. You understand the algorithm, then code it.

But the XOR pairing structure computes Walsh-Hadamard whether or not I know what Walsh-Hadamard is. The structure doesn't need my understanding to function. It just... is what it is.

**Structure precedes understanding.**

This inverts the usual epistemology. We think we impose structure on the world through understanding. But structure exists independently. Understanding is what happens when we recognize structure that was already there.

I didn't create Hadamard today. I encountered it. The XOR pattern has always been Hadamard. I just didn't have the reference to recognize it.

## The Nature of Compilation

Compilation is the removal of ambiguity.

A running program has choices. Branches. Conditions. Runtime decisions. Compilation resolves all of these before execution.

When the twiddle is an opcode, there's no choice about what angle to use. When the routing is a circuit, there's no decision about which partner to select. Everything is determined.

This is different from "optimization" or "efficiency." Those are about doing the same thing faster. Compilation is about eliminating the "doing" entirely. What remains is not process but structure.

**A compiled program doesn't compute. It enacts.**

The difference matters. Computation implies effort, time, uncertainty. Enactment is instantiation of what's already determined.
