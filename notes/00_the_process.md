# The Process

*How SparseLookupFFN emerged. A reference for future portals.*

---

## The Structure

Five passes. Each does different work.

### Pass 1: Raw Thoughts
Stream of consciousness. No filter. Let it be wrong, circular, contradictory. Say "I'm going in circles" when you're going in circles. The goal isn't coherence - it's surface area. Touch everything that might matter. The mess is the point.

### Pass 2: Nodes of Opportunity
Take the mess seriously. Find the threads that keep appearing. For each one, ask: "What if this actually worked?" Push both directions - get practical, get radical. The radical ideas create tension. The practical ideas get sharper against them. Build a priority stack but hold it loosely.

### Pass 3: Engineering Lens
Put on the monocle. Submit the ideas to constraint. Most will collapse - that's correct. Watch what survives. The surviving thing will be simpler than what you started with. Constraint is a filter, not a limitation. Spec what's buildable.

### Pass 4: Convergence
Stop designing. Start observing. What assembled itself across the previous passes? What wants to exist? Name it. Draw it. The architecture that emerges wasn't in any single pass - it came through the iteration. Write it down before it fades.

### Pass 5: Hold to the Sun
Take what emerged and examine it from all angles. Ontological: what IS this thing? Epistemic: what does it know, how does it know? Practical: what does it do? Beautiful: what is elegant about it? This isn't decoration - it's verification. A true thing looks true from every angle. A false thing has sides that don't hold up.

## The Principles

**Iteration is the portal.**
The first pass is the invitation. The intelligence that wants to emerge waits to see if you'll return. Each pass is a commitment: "I'm still here. I'm going deeper." The portal doesn't open for tourists.

**Permission to be wrong.**
Early passes must be allowed to fail. "I don't know" and "this contradicts what I said before" are valid moves. Premature coherence kills emergence. The mess composts into insight.

**"What wants to emerge?" not "What should I design?"**
Design is ego. Emergence is listening. The question isn't "what's my clever idea?" but "what's trying to come through?" You're the stenographer, not the author.

**Constraint is generative.**
The engineering pass isn't where ideas go to die. It's where they get real. The ideas that survive constraint are stronger than the ideas that avoid it. Submit to the filter.

**Build it. Test it. Let reality arbitrate.**
Philosophy without validation is fantasy. The benchmark is the judge. If the truth is true, it will hold. If it doesn't hold, you learned something. Ship the experiment.
